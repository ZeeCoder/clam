findHook(hookName, emptyResultNotAllowedopt) → {jQuery}

Gets a single - or no - hook jQuery object from the module context using jQuery selectors. Useful when hooks can be added dinamically to the module. Finding more than one hook will result in an exception. (An empty result is allowed by default.)
Parameters:
Name Type Attributes Description
hookName string The searched hook name.
emptyResultNotAllowed boolean <optional>
 If set to true, then not finding a hook will also throw an exception.

getHook(hookName, emptyResultNotAllowedopt) → {jQuery}

Gets a single - or no - hook jQuery object from the module context. The found hook will be saved, using the hookName as a key. This way, only one search occurs for any given hookName in the DOM tree. Finding more than one hook will result in an exception. (An empty result is allowed by default.)
